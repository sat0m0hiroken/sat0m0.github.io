【HTML/CSS】CSSセレクタのおさらい - セレクタ

* (ユニバーサルセレクタ)

See the Pen Selector Exsample (* - Universal selector) by sksksk (@__sk__) on CodePen.

全てのセレクタに適応できます。ユニバーサルセレクタとも呼ばれます。
「div *」とすると、divの子要素全てという指定になります。
優先度が「!important」の次に高いので利用する時は注意しましょう。

. (classセレクタ)

See the Pen selector(class) by sksksk (@__sk__) on CodePen.

指定したクラス名だけにCSSが適応されます。
「p.sample」とすると、クラス名「sample」が付いたP要素のみに適応されます。
もしP要素からa要素にHTML構造を変更すると適応されません。気をつけましょう。

# (idセレクタ)

See the Pen selector exsample(ID) by sksksk (@__sk__) on CodePen.

使い方はclassと同じですが、使い回しはできません。
CSS反映の優先度が高いのでclassと併用する時は注意しましょう。

その他セレクタ指定方法

See the Pen CSSセレクタ01 by sksksk (@__sk__) on CodePen.

Parent Child （指定した親要素の子要素に適用する）
Parent > Child （指定した親要素の直下の子要素に適用する）
Element + Element （隣接する要素に適用）
Element ~ Element （【CSS3】後ろにある要素に適用）

「Element + Element（隣接する要素に適用）」の1番最初のセレクタだけなぜCSSが適用されていないのか、
厳密には隣接セレクタと言っても、「最初に指定した要素」 + 「その次に指定する要素」からCSSが適用対象となるからです。
例えば「Aセレクタ + Bセレクタ」とした場合、Aセレクタの直後にあるBセレクタが対象になります。
