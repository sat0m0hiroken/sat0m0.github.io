DEBUG

デバッグ方法と流れ

作業前

依頼内容を洗い出し、リストアップをします。
必要であればイメージデータのスライスをします。

作業開始～完了

新規作成ページ以外はFTP上からバックアップを必ず取りましょう。

cssやjsはHTML内に直接記述せず、外部ファイルとして読み込ませましょう。

コーディング完了後デバッグを行う。
デバッグ後、依頼者へ確認。

依頼中に確認してもらう用のサーバーのURLと、本番環境へ適応する用のデータを2種用意します。直ぐにエンジニアへ依頼できるよう外部ファイル・imgのパス変更を行う等、予め準備しておくとスムーズです。

エンジニアへphp化の依頼をかけます。

本番サーバーへCSSをUPしておきましょう。
HTMLファイルは不要です。

作業完了後、LPの場合は作業に使用したデータ・作成したイメージデータを共有フォルダにUPします。

デバッグの大まかな流れ

見た目・動きチェック
依頼内容と照らし合わせ、抜け漏れが無いかチェックします。
構文チェック・ブラウザチェック
構文やファイルに対して問題点が無いか抽出し、問題点があれば修正しましょう。
https://validator.w3.org/
リンク切れがないかチェックします。
https://validator.w3.org/checklink
画像のaltが最適についているか、width,heightが書かれているか確認します。
ここまで特に問題がなければ、最後にGoogleでも確認します。
https://developers.google.com/speed/pagespeed/insights/
必要に応じてHTMLを整形するツールを使いましょう。
https://dirtymarkup.com/
その後各主要ブラウザで表示チェックを行います。
PC(Win) : Chrome,Fire Fox,IE11,IE Edge
PC(Mac) : Safari,Chrome
SP(iPhone) : Safari,Chrome
SP(Android) : Safari,Chrome
